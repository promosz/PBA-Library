# Jak używać PBA Library

Witaj w bibliotece komponentów PBA! Ta strona zawiera wszystkie potrzebne informacje do rozpoczęcia pracy z komponentami.

## 🚀 Uruchomienie Storybook

```bash
npm run storybook
```

Storybook będzie dostępny na: **http://localhost:6007** (lub innym porcie jeśli 6006 jest zajęty)

## 📁 Struktura projektu

```
PBA-Library/
├── .storybook/           # Konfiguracja Storybook
├── src/
│   ├── components/       # Komponenty z Figma
│   │   └── Button/       # Przykładowy komponent
│   └── styles/           # Globalne style
├── stories/              # Przykłady Storybook
└── package.json          # Skrypty i zależności
```

## 🎨 Dodawanie komponentów z Figma

### 1. Przygotowanie komponentu

1. **Stwórz folder komponentu:**
```bash
mkdir src/components/YourComponent
```

2. **Dodaj pliki:**
- `YourComponent.tsx` - React component
- `YourComponent.css` - Style
- `YourComponent.stories.tsx` - Storybook stories
- `index.ts` - Export

### 2. Template komponentu

```tsx
import React from 'react';
import './YourComponent.css';

export interface YourComponentProps {
  children?: React.ReactNode;
  className?: string;
}

export const YourComponent: React.FC<YourComponentProps> = ({
  children,
  className = '',
  ...props
}) => {
  return (
    <div className={`pba-your-component ${className}`} {...props}>
      {children}
    </div>
  );
};
```

### 3. Template story

```tsx
import type { Meta, StoryObj } from '@storybook/react';
import { YourComponent } from './YourComponent';

const meta: Meta<typeof YourComponent> = {
  title: 'PBA Components/YourComponent',
  component: YourComponent,
  parameters: {
    layout: 'centered',
    // Dodaj link do Figma
    design: {
      type: 'figma',
      url: 'https://www.figma.com/file/YOUR_FILE_ID/YOUR_FILE_NAME?node-id=NODE_ID',
    },
  },
  tags: ['autodocs'],
};

export default meta;
type Story = StoryObj<typeof meta>;

export const Default: Story = {
  args: {
    // Default props
  },
};
```

## 🎯 Design System

### Kolory PBA

```css
:root {
  --pba-primary: #1e40af;      /* Główny kolor PBA */
  --pba-secondary: #3b82f6;    /* Kolor pomocniczy */
  --pba-accent: #f59e0b;       /* Kolor akcentowy */
  --pba-success: #10b981;      /* Sukces */
  --pba-warning: #f59e0b;      /* Ostrzeżenie */
  --pba-error: #ef4444;       /* Błąd */
  --pba-text-primary: #1f2937; /* Tekst główny */
  --pba-text-secondary: #6b7280; /* Tekst pomocniczy */
  --pba-background: #ffffff;   /* Tło */
  --pba-surface: #f9fafb;     /* Powierzchnia */
  --pba-border: #e5e7eb;      /* Ramka */
}
```

### Typografia

- **Font family:** System fonts (San Francisco, Segoe UI, etc.)
- **Font weights:** 400 (normal), 500 (medium), 600 (semibold)

### Spacing

- **Small:** 0.5rem (8px)
- **Medium:** 1rem (16px)
- **Large:** 1.5rem (24px)
- **XL:** 2rem (32px)

## 🔗 Integracja z Figma

### Jak dodać link do Figma:

1. **Otwórz plik Figma**
2. **Wybierz komponent/ramkę**
3. **Kliknij "Share" → "Copy link"**
4. **Dodaj URL do story:**

```typescript
parameters: {
  design: {
    type: 'figma',
    url: 'https://www.figma.com/file/FILE_ID/FILE_NAME?node-id=NODE_ID',
  },
}
```

## 📝 Najlepsze praktyki

### 1. Nazewnictwo
- Używaj prefiksu `pba-` dla klas CSS
- Nazwy komponentów w PascalCase
- Nazwy plików w PascalCase

### 2. Dostępność
- Dodawaj odpowiednie atrybuty ARIA
- Testuj nawigację klawiaturą
- Używaj semantycznych elementów HTML

### 3. Responsywność
- Projektuj mobile-first
- Testuj na różnych rozmiarach ekranu
- Używaj CSS Grid i Flexbox

### 4. Dark Mode
- Wspieraj ciemny motyw używając CSS variables
- Testuj komponenty w obu motywach
